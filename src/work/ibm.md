challenge: "Each client had different starting points — legacy on-premise workloads, inconsistent deployment practices, security debt, and limited cloud governance. The challenge was to deliver cloud migration and modernisation at enterprise scale while maintaining security, compliance, and operational consistency across three cloud providers."
approach: "Worked as a DevOps engineer and lead within IBM's Azure Cloud Practice across four concurrent client engagements. For H&M: drove cloud migration and security remediation for 100+ applications, enforcing tiered SLA remediation (High: 8 hrs, Medium: 48 hrs, Low: 7 days) via SonarQube and Coverity; provisioned Azure landing zones via Terraform; defined migration approach — lift-and-shift, modernization, and infrastructure upscale — alongside app owners; executed SQL-based data migration and migrated application code to GitHub. For Siemens: provisioned AWS Terraform landing zone and GitHub Actions pipelines; released containerized applications into AKS namespaces via Helm and ACR; secured all deployment workflows with OIDC authentication. For Südzucker: provisioned GCP landing zone via Terraform and GitHub Actions. For Nokia: supported Azure migration for a US government-aligned client under active security clearance (clearance expired 2024); delivered compliance-driven infrastructure under regulated enterprise requirements."
